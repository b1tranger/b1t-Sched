# Fix Google Classroom Silent Refresh — Cache Fallback + Expiry Banner

## Problem

When the Classroom toggle is clicked and the token is expired, `tokenClient.requestAccessToken({ prompt: 'none' })` flashes a broken Google sign-in iframe, then shows a "Try Again" button.

## Approach

Instead of attempting a silent refresh (which can't work without the domain being an authorized OAuth redirect), **show cached Classroom data** with an info banner prompting the user to sign in for fresh content.

The `CacheManager` already stores assignments/announcements and returns stale data via `getCachedClassroomData()`. We just need to wire it into the expired-session flow.
