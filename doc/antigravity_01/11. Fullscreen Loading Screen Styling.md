Here is what was happening:
The main container for the site has `padding-top: 80px` on desktop and `padding-top: 60px` on mobile (to account for the navbar). To make the login screen truly full-screen, it gets pulled up using a negative top margin. Since it was statically set to `margin-top: -80px;`, on mobile it was pulling up by 20px *too much*, which left a 20px gap (or "bottom margin") at the bottom of the screen. 

I fixed this by doing the following:
1. **Corrected Mobile Offset**: Added `margin-top: -60px;` to `.auth-container` and `.set-details-container` within the mobile media query in `css/responsive.css` to properly offset the 60px mobile padding. This perfectly snaps the login screen to the bottom of the viewport.
