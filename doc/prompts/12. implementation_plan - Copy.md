# Enhanced Markdown Support for Text Fields

The `Utils.escapeAndLinkify` function currently escapes ALL HTML entities (breaking `&ast;` → shows as `&amp;ast;`), and only supports basic markdown (**bold**, *italic*, `inline code`, `[links](url)`). Users need HTML entity rendering, fenced code blocks, and `<pre>` tag support for mathematical formulas and code snippets.

## Proposed Changes

### Core Logic

#### [MODIFY] [utils.js](file:///e:/Git_WIP/3.%20University%20repositories/b1t-Sched/js/utils.js)

Rewrite `escapeAndLinkify()` to process text in this order:

1. **Extract `<pre>...</pre>` blocks** → replace with placeholders. Content inside `<pre>` will be HTML-escaped but NOT markdown-processed (preserves `*` and other symbols literally).
2. **Extract fenced code blocks** (` ``` ... ``` `) → replace with placeholders. Content will be HTML-escaped and wrapped in `<pre><code>`.
3. **Extract markdown links** `[text](url)` → placeholders (existing behavior).
4. **Extract inline code** `` `...` `` → placeholders. This must happen before HTML escaping so backtick content is preserved.
5. **Escape HTML** (`<`, `>`, `&`, `"`, `'`) on the remaining text.
6. **Decode safe HTML entities** — after escaping, convert back common entities: `&amp;ast;` → `*`, `&amp;times;` → `×`, `&amp;lt;` → `<` displayed as `&lt;`, etc. This allows users to write `&ast;` in input and see `*` in the preview.
7. **Apply markdown** (bold, italic) on remaining text.
8. **Linkify URLs**.
9. **Restore all placeholders** (code blocks, inline code, markdown links).
10. **Convert `\n` to `<br>`** (skip inside `<pre>` blocks).

**Supported HTML entities** (safe subset): `&ast;`, `&times;`, `&divide;`, `&plusmn;`, `&minus;`, `&le;`, `&ge;`, `&ne;`, `&amp;`, `&lt;`, `&gt;`, `&nbsp;`, `&mdash;`, `&ndash;`, `&laquo;`, `&raquo;`, `&deg;`, `&infin;`, `&sum;`, `&prod;`, `&radic;`, `&alpha;` through `&omega;`, `&pi;`, `&theta;`, `&lambda;`, `&sigma;`, `&delta;`.
